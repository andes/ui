Listados

Ordenamiento dentro del layout

La secuencia de componentes que debe encontrar el usuario es:
1. Pestañas (en caso de haber)
2. Batería de filtros
3. Subtítulo (en caso de ser necesario) junto a los botones que operan sobre el listado completo o una selección múltiple.
4. Listado

Jerarquización

Para evitar saturación de información, se recomienda hacer un análisis previo mediante el cual se identifique y diferencie la información principal de la información secundaria, y a su vez, de aquella que el usuario puede visualizar interacción mediante (por ej. en un panel auxiliar o un dropdown).

Item de la lista

Dentro del listado, el contenido de cada item se ordena de izq. a der. de la siguiente manera:


Elementos marginados a la izquierda:
1. Checkbox (en caso de haber)
2. Icono (en caso de haber)
3. Título (Dependerá del módulo o finalidad de la pantalla)
4. Subtítulo (Información secundaria: Nombre de profesional, datos de organización, datos secundarios de paciente)

Elementos marginados a la derecha:
1. Badge (estados, fechas, advertencias)
2. Acciones (textuales e/o iconográficas)
a. Acciones sensibles
b. Acciones de neutras (actualización, modificación)
c. Acciones de confirmación
d. Checkbox (en caso de ser seleccionable)
3. Dropdown
a. aloja opciones secundarias (por ej.: ver detalle)

Importante! En pantallas de menor dimensión (> 780px), las acciones se jerarquizan, identificando la acción principal y relegando el resto de los elementos se solapan en el dropdown.

Mobile

  • Se secuencian las pantallas
  • El friso se mantiene fijo mientras se scrollea el contenido
  • En caso de haber pestañas, estarán visibles en tanto el scroll esté en posición inicial. Si el usuario comienza a interactuar con los elementos, las pesatañas hacen fade-out. (se entiende que el usuario desea estar en dicha pestaña).
