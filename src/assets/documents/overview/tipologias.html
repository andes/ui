Tipologías

El sistema de interfaces se genera a partir de diferentes esquemas de ordenamiento:

Estándar

Presenta un panel claramente dominante sobre el otro.

Cuándo se recomienda utilizar

Cuando debo crear, visualizar o modificar elementos que se alojan en el panel principal.
Ej.: Actualizar la información de un usuario o arrastrar una prestación para ser registrada.

Equilibrado

Se subdivide la pantalla en dos campos de igual dimensión.

Cuándo se recomienda utilizar

Cuando el volumen y la calidad de la información en ambos paneles es igualmente relevante.
Ej.: Comparación de conflicto de datos entre usuarios MPI.

Pantalla completa

Se compone de un sólo panel que abarca la totalidad de la pantalla.

Cuándo se recomienda utilizar

Cuando la operación que está siendo ejecutada requiere que el usuario este sólo enfocado en una pantalla.
Ej.: visualización de un informe o un resumen de consulta.
